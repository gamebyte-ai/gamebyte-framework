# v1.8 Correctness Fixes Plan

> Gamelabs.js audit sonucu: 8 kritik bug + 6 design gap + 2 önceki sorun

## Paralel Gruplar

### Grup A: Core Framework (ServiceContainer + GameByte + BaseScene)

**ServiceContainer.ts:**
1. Circular dependency detection — factory çağrısı sırasında `_creating` flag
2. Duplicate binding uyarısı — `bind()` zaten varsa `Logger.warn` 
3. `instance()` double-register fix — `this.instances.set()` yeter, `singleton()` çağırma

**GameByte.ts:**
4. `static instance` destroy'da null'la — `GameByte.instance = null as any`
5. `boot()` emit timing — `provider:booted` tüm boot'lar bittikten sonra emit et
6. `destroy()` emit sırası — `emit('destroyed')` en sona al (cleanup sonrasına)

**BaseScene.ts:**
7. `as unknown as PIXI.Container` kaldır — `IContainer` olarak tut, cast etme

### Grup B: Camera + AssetManager

**Camera.ts:**
8. Frame-rate-independent lerp — `1 - Math.exp(-k * dt)` formülü
9. Zoom lerp de aynı fix

**GameByteAssetManager.ts:**
10. `loadAssetEntry()` await ekle
11. `setInterval` kaldır — processQueue() zaten load sonrası kendini çağırır
12. Timeout/resolve race fix — timeout'ta asset store etme
13. `loadBundle()` dead code uyarısı

### Grup C: Screen/Panel Lifecycle

**ScreenManager.ts:**
14. `animateTransition()` etrafına try/catch + finally `isTransitioning = false`
15. Screen transition'da destroyed guard

**PanelManager.ts:**
16. `resize()` da `initialize()` yerine `resize()` çağır
17. Panel transition sırasında `interactiveChildren = false`

### Grup D: Design Patterns

**UnsubscribeBag utility (yeni dosya):**
18. `src/utils/UnsubscribeBag.ts` — LIFO cleanup, try/catch per callback, idempotent flush

**SettingsPanel upgrade:**
19. Typed fields: boolean + number (min/max/step)
20. Auto-persist: SaveSystem veya localStorage entegrasyonu
21. Number field için slider/stepper UI

## Bağımlılıklar

- A, B, C, D birbirinden bağımsız → 4 paralel agent
- Grup D (UnsubscribeBag) diğer gruplarda kullanılabilir ama zorunlu değil
